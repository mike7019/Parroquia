### 1. **Carga Automática de Variables desde .bashrc** ✨

   - Genera archivo `.env.docker` automáticamente
   - No necesitas configurar variables manualmente

## 🚀 Inicio Rápido

### Conectar al Servidor Linux
```bash
# Conectar por SSH
ssh -p 2223 l4bsb@206.62.139.100

# Ir al directorio del proyecto
cd /home/l4bsb/Parroquia
```

### Paso 1: Configurar Variables
```bash
# Verificar que las variables están en .bashrc (ya las tienes)
grep "export DB_" ~/.bashrc

# Recargar (si hiciste cambios)
source ~/.bashrc
```

### Paso 2: Hacer Scripts Ejecutables
```bash
chmod +x deploy-docker.sh docker-utils.sh verify-docker-setup.sh
```

### Paso 3: Verificar
```bash
./verify-docker-setup.sh
```

### Paso 4: Desplegar
```bash
./deploy-docker.sh --build
```

### Paso 5: Verificar
```bash
./docker-utils.sh health
curl http://localhost:3000/api/health
```

## 📊 Variables de Entorno Soportadas

Según tu imagen de `.bashrc`, el sistema carga automáticamente:

✅ **Base de Datos**
- DB_HOST, DB_PORT, DB_NAME, DB_USER, DB_PASSWORD

✅ **JWT**
- JWT_SECRET, JWT_REFRESH_SECRET, JWT_EXPIRES_IN, JWT_REFRESH_EXPIRES_IN

✅ **Email / SMTP**
- EMAIL_HOST, EMAIL_PORT, EMAIL_USER, EMAIL_PASS, EMAIL_FROM
- SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS
- SEND_REAL_EMAILS, SMTP_FROM_EMAIL

✅ **Aplicación**
- NODE_ENV, PORT, APP_NAME, APP_URL, FRONTEND_URL

✅ **Seguridad**
- CORS_ORIGIN, RATE_LIMIT_WINDOW_MS, RATE_LIMIT_MAX

✅ **Logging**
- LOG_LEVEL

✅ **API Keys** (opcional)
- GEMINI_API_KEY, GITHUB_TOKEN

## 🔥 Comandos Más Usados

```bash
# Desplegar / Actualizar
./deploy-docker.sh --build

# Ver logs en tiempo real
./docker-utils.sh logs

# Ver estado
./docker-utils.sh status

# Backup de BD
./docker-utils.sh backup

# Reiniciar API
./docker-utils.sh restart api

# Entrar a contenedor
./docker-utils.sh shell api

# Ver salud
./docker-utils.sh health
```
